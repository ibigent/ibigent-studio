# Contributing

Thank you for helping improve the IBIGENT Studio documentation.

## What we accept
- Fixes to typos, broken links, unclear steps, and translations (EN/KO).
- Documentation must match the official docs: https://www.ibigent.com/docs/ibigent-studio/

## What we do not accept
- Product source code.
- Any secret, API key, password, or personal data.
- Unverified configuration for unsupported tools (mark as TODO instead).

## How
1. Fork and create a branch.
2. Make focused documentation edits.
3. Open a pull request describing the change.
